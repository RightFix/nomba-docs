Unofficial · community-maintained

Every Nomba endpoint, already wired.

nomba is a Python SDK generated directly from Nomba's own OpenAPI spec — 64 endpoints across 10 resource groups, sync and async, with typed responses and request validation built in. No endpoint URLs to memorize, no manual token handling, no guessing field names. 

pip install nomba
# or
uv add nomba
from nomba import Nomba

nomba = Nomba(
    client_id="...",
    client_secret="...",
    account_id="...",
    sandbox=True,
)

account = nomba.virtual_accounts.create_virtual_account(
    account_ref="ref-123",
    account_name="Jane Doe",
)

What's covered

64 endpoints Full API reference Accounts, Virtual Accounts, Checkout, Charge, Transfers, Terminals, Transactions, Airtime/Data, CableTV, Electricity. Sync + async Getting started Auth, error handling, and the shape of every response. Cursor-based Pagination iterators Loop over every page of a list endpoint without writing the loop yourself. 3 steps Guided card payment flow Card details → OTP or 3DS → confirm, decoded into plain booleans. HMAC-SHA256 Webhook verification Signature check plus replay-window protection on the timestamp. Locked + retried Reliability built in Token-fetch locking and automatic backoff on 429/5xx responses.

Why generated, not hand-written

Every resource method, its required and optional fields, and its response shape come straight from Nomba's published OpenAPI spec — not transcribed by hand. That means field names match exactly (accountRef, not a guess at account_ref), and nested request bodies are checked locally against the same spec before anything goes over the network.

Scope

This SDK is unofficial and not affiliated with Nomba. It wraps Nomba's public API; see developer.nomba.com for the canonical reference and your dashboard credentials.
